# Deferred features (Tier C & “never for small Skyfall”)

**Purpose:** Record features we intentionally skip for a lean Skyfall scope so agents and design notes do not re-open them without a product decision.

**Source tier:** [SKYFALL_LEAN_FEATURES.md](./SKYFALL_LEAN_FEATURES.md) — **Tier C** and anti-bloat principles (second economies, gacha, passes, calendars).

**Formula lens:** [POPULAR_SIMPLE_GAMES_FORMULA.md](./POPULAR_SIMPLE_GAMES_FORMULA.md) — many patterns *can* work in hits, but **§5 Caveats** applies: correlation ≠ causation; ethics around variable rewards; **platform** mixes design with UA, monetization, and live ops. Deferrals here are **scope and identity** choices, not claims that these patterns never succeed elsewhere.

---

| Deferred item | One-line rationale | Revisit when |
|---------------|-------------------|--------------|
| **Idle / offline earnings** | New economy and passive-play expectation; fights the core “active dodge” fantasy. | You ship an idle/incremental SKU or explicitly want check-in progression (see formula §1D incremental branch). |
| **Multiple currencies + vendors** | Cognitive load and balancing surface explode vs. one meta pipe. | Mobile F2P or deep meta economy with a dedicated balance owner and UI budget. |
| **Procedural narrative / run quests beyond contracts** | Writing, branching, and tooling cost; overlaps poorly with existing daily contracts. | Narrative-led release or story mode is the primary pitch, not a patch on the dodger. |
| **Full cosmetic shop** | Asset pipeline + shop UI + entitlement work; OK as **Phase 2** expression, not a small patch. | Phase 2 cosmetics/monetization is scoped (art, shop UX, save rules). |
| **Real-time leaderboards / async PvP** | Backend, moderation, cheat/fairness across builds and versions. | You **own** infra and ops and want competitive/social proof (formula caveats: platform + fairness expectations). |
| **Roguelike “full relic grid”** | Becomes a build-craft game (e.g. *Hades*-scale), not tightening Skyfall’s loop. | Product pivots so run-long build systems are the main verb, with budget to match. |
| **Gacha chests** (optional anti-bloat) | Variable-reward economy + ethics surface (formula §1E, §5 ethics). | Mobile F2P SKU with explicit monetization design and compliance review. |
| **Battle passes** (optional anti-bloat) | Seasonal entitlement + content calendar; not a solo patch. | Dedicated live-ops or hybrid-casual SKU (formula §5 platform). |
| **Live ops calendars** (optional anti-bloat) | Requires ongoing scheduling, events, and comms—not “small Skyfall.” | Same as passes: team + mobile/hybrid product with ops capacity. |
